# Job Search Workflow

A practical end-to-end workflow for using Resume Engine throughout an active job search.

## Setup (Once)

```bash
# 1. Install
pip install resume-engine

# 2. Set up Ollama (free, local)
ollama pull qwen2.5:14b
ollama serve

# 3. Create or import your master resume
resume-engine import --text old-resume.txt --output master-resume.md
# -- or write master-resume.md from scratch --

# 4. Create your workspace structure
mkdir -p ~/job-search/{jobs,applications}
```

## Per-Job Workflow

For each job you want to apply to:

### 1. Save the job posting

```bash
# Paste the full job description into a file
nano ~/job-search/jobs/company-role.txt

# Or scrape it (if the URL is accessible)
# resume-engine tailor can scrape with --job-url
```

### 2. Check your baseline ATS score

```bash
resume-engine ats \
  --resume ~/job-search/master-resume.md \
  --job ~/job-search/jobs/company-role.txt
```

If you're below 30%, consider whether this role is worth applying to -- the gap may be too large to bridge.

### 3. Tailor the resume

```bash
resume-engine tailor \
  --master ~/job-search/master-resume.md \
  --job ~/job-search/jobs/company-role.txt \
  --output ~/job-search/applications/company-role/resume.md \
  --model ollama
```

### 4. Check your tailored ATS score

```bash
resume-engine ats \
  --resume ~/job-search/master-resume.md \
  --job ~/job-search/jobs/company-role.txt \
  --tailored ~/job-search/applications/company-role/resume.md
```

Aim for 70%+ before submitting. If you're under that:
- Check the "Still missing" keywords
- Add them manually if they genuinely apply to your background
- Re-run `tailor` with `--model openai` for a higher-quality pass

### 5. Generate the cover letter

```bash
resume-engine cover \
  --master ~/job-search/master-resume.md \
  --job ~/job-search/jobs/company-role.txt \
  --output ~/job-search/applications/company-role/cover-letter.md
```

### 6. Export to PDF (optional)

```bash
# Convert markdown to PDF
pandoc ~/job-search/applications/company-role/resume.md \
  -o ~/job-search/applications/company-role/resume.pdf

# Or use resume-engine's built-in PDF export from the start:
resume-engine package \
  --master ~/job-search/master-resume.md \
  --job ~/job-search/jobs/company-role.txt \
  --outdir ~/job-search/applications/company-role/ \
  --format pdf
```

## Batch Workflow (Active Job Search)

When applying to many jobs at once, use batch mode:

```bash
# Structure
~/job-search/
  master-resume.md
  jobs/
    stripe-swe.txt
    google-platform.txt
    datadog-backend.txt
  applications/   (created by batch)

# Run batch
resume-engine batch \
  --master ~/job-search/master-resume.md \
  --jobs-dir ~/job-search/jobs/ \
  --outdir ~/job-search/applications/ \
  --with-cover \
  --model ollama
```

This generates a full application package for each company in ~30-60 seconds per job.

## Directory Structure

```
job-search/
  master-resume.md          # Your master resume (keep this up to date)
  jobs/                     # Job postings (one file per job)
    stripe-swe.txt
    google-swe.txt
  applications/             # Generated by resume-engine
    stripe-swe/
      resume.md
      cover-letter.md
      resume.pdf
    google-swe/
      resume.md
      cover-letter.md
```

## Quality Checklist

Before submitting any application:

- [ ] ATS score is 70%+ on the tailored resume
- [ ] Resume reads naturally (review for awkward AI phrasing)
- [ ] Dates are accurate (LLM might have altered them)
- [ ] Contact info is correct
- [ ] No unexplained gaps in employment history
- [ ] Cover letter mentions the company name (not "[Company]")
- [ ] PDF renders cleanly (check formatting, no broken layouts)
